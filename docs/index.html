IBM WatsonX Challenge

Oil & Gas
Opportunity Intelligence Assistant

An AI-powered Slack assistant that helps IBM teams working with oil and gas clients quickly understand industry developments, identify opportunities, and take action.

Get Started View on GitHub

Key Features

Scheduled Intelligence

Every 10 hours the bot generates structured updates — news, market conditions, account insights, IBM relevance, and recommended next steps.

Real-Time Deal Radar

Continuous scanning for major client-related events with proactive Slack alerts for urgent developments and high-value opportunities.

IBM Solution Mapping

Automatically connects market events to relevant IBM offerings: watsonx, Maximo, Consulting, Hybrid Cloud, Cybersecurity, and more.

Workflow Automation

After identifying opportunities, the assistant drafts outreach, creates meeting briefs, logs leads in CRM, and schedules follow-ups.

AI Agent Architecture

Market Monitor AgentTracks crude oil prices, natural gas, rig counts, LNG trends, OPEC+ activity
News, Policy & Emissions AgentMonitors regulatory developments, sustainability trends, and policy changes
Client Intelligence AgentAnalyzes account-specific data for ExxonMobil, Chevron, SLB, Halliburton, Shell, BP, ConocoPhillips
Credibility & Deduplication AgentValidates sources and removes duplicate content before delivery
Deal Radar AgentScores opportunities by client relevance, urgency, IBM fit, and executive visibility
Workflow Automation AgentExecutes follow-up actions: outreach drafts, CRM logging, briefing scheduling

Getting Started

Prerequisites

  • Node.js 18+
  • Slack workspace with bot permissions
  • API keys for news and market data sources
  • IBM watsonx access

Installation

# Clone the repository
git clone git@github.com:IBMAyush/WatsonXChallenge.git
cd WatsonXChallenge

# Install dependencies
npm install

# Configure environment variables
cp .env.example .env
# Edit .env with your API keys

# Run the application
npm start

Environment Variables

VariableDescription
SLACK_BOT_TOKENSlack bot token (starts with xoxb-)
SLACK_APP_TOKENSlack app-level token (starts with xapp-)
WATSONX_API_KEYIBM watsonx API key
NEWS_API_KEYNews data source API key
MARKET_DATA_API_KEYMarket data source API key

Running Tests

# Run all tests
npm test

# Run specific suite
npm test -- agents

# Run with coverage
npm run test:coverage

Slack Commands

Once the bot is running and added to your Slack workspace, use these commands:

/oilgasbot any relevant news regarding ExxonMobil?
/oilgasbot give me current oil and gas monitor
/oilgasbot prep me for a meeting with Halliburton
/oilgasbot show top oil and gas opportunities this week
/oilgasbot alert me on major Chevron updates

Git Workflow for the Team

Everyone on the team should follow this workflow to keep the repo clean and conflict-free.

2
Always pull before starting workgit pull origin main — gets the latest changes from everyone else.
3
Create a feature branchgit checkout -b feature/your-feature-name — never commit directly to main.
4
Stage and commit your changesgit add . then git commit -m "Add feature: clear description"
5
Push your branchgit push -u origin feature/your-feature-name
6
Open a Pull Request on GitHubGo to github.com/IBMAyush/WatsonXChallenge, open a PR from your branch into main, get it reviewed, then merge.

Troubleshooting

If your push is rejected, someone else pushed first. Run git pull origin main, resolve any conflicts, then push again. If you need help with SSH setup, check that ~/.ssh/id_ed25519.pub exists and is added to your GitHub account under Settings → SSH Keys.
